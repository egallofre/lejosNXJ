Conexión entre dispositivos Android y Robots Lego Mindstorms NXT mediante Bluetooth

En este tutorial vamos a ver cómo se puede controlar un robot de la firma Lego, concretamente los robots de Lego MINDSTORM, con un dispositivo Android.

Para resolver la conexión entre el dispositivo móvil Android y el robot NXT implementaremos un canal de comunicación o socket bluetooth entre dos aplicaciones Java, una instalada en el robot y que actuará como servidor y otra instalada en el dispositivo Android y que actuará como cliente.

El NXT es el “cerebro” de los robots Lego MINDSTORMS. Es un bloque Lego que contiene el microprocesador que permite controlar los robots y utilizar sensores y actuadores, en definitiva, darles vida. Los ladrillos NXT poseen tres puertos para motores (A, B y C), cuatro puertos para sensores (1, 2, 3 y 4), un puerto USB, conexión inalámbrica Bluetooth, pantalla, varios botones para su control y una bocina.

El NXT permite utilizar Java para desarrollar programas de control, lo cual aporta mucha flexibilidad con respecto a la utilización del software gráfico proporcionado por Lego MINDSTORMS. Para ello se emplea leJOS NXJ, que además requiere la actualización del firmware NXT  para instalar en él la JVM. La actualización del firmware puede llevarse a cabo directamente con el plugin de LeJOS para Eclipse http://lejos.sourceforge.net/tools/eclipse/plugin/nxj/.

Existen diversas librerías para la creación de programas Java para el NXT, entre las más utilizadas están la propia librería de LeJOS NXJ http://lejos.sourceforge.net/nxt/nxj/api/index.html o la NXTJLib http://www.aplu.ch/home/apluhomex.jsp?site=27.

Para la comunicación entre ambas aplicaciones, tanto la aplicación del robot como la del dispositivo Android deben compartir un protocolo de comunicación. Este protocolo estará compuesto por las órdenes que el dispositivo Android puede enviar al robot y que este puede interpretar y ejecutar cuando las recibe.

public boolean conectarBT(){
    btc = Bluetooth.waitForConnection(0, NXTConnection.RAW);
 
    if (btc!=null){
        dis = btc.openDataInputStream();
        dos = btc.openDataOutputStream();
    }
 
    return (btc!=null);

- DEL LADO DEL DISPOSITIVO ANDROID - INTERACCIÓN

El dispositivo será el cliente de la comunicación bluetooth. De él partirán las órdenes que el usuario indique y que determinarán el comportamiento del robot.

Para crear el socket bluetooth con el robot se realizará una búsqueda de dispositivos bluetooth disponibles hasta localizar el NXT y conectarse a él. Para que la conexión pueda llevarse a cabo, previamente el dispositivo Android ha debido vincularse con el robot NXT.

// Elementos del paquete bluetooth necesarios
import android.bluetooth.BluetoothAdapter;
import android.bluetooth.BluetoothDevice;
import android.bluetooth.BluetoothSocket;
 
// Conexión punto a punto --> canal de comunicación
private BluetoothSocket nxtBTsocket = null;
 
// Punto de salida del canal de comunicación
// Se usa para enviar hacia el robot
private DataOutputStream nxtDos = null;
 
// Punto de entrada del canal de comunicación
// Se usa para recibir desde el robot
private DataInputStream nxtUno = null;
 
// Creación del canal de comunicación entre dispositivo Android y Robot
private void createNXTConnection(){
   Toast toast;
   try {
      // Obtenemos el adaptador bluetooth
      BluetoothAdapter btAdapter = BluetoothAdapter.getDefaultAdapter();
 
      // Recorremos la lista de dispositivos descubiertos                
      Set<BluetoothDevice> bondedDevices = btAdapter.getBondedDevices();
      BluetoothDevice nxtDevice = null;
      for (BluetoothDevice bluetoothDevice : bondedDevices)  {
         if (bluetoothDevice.getName().equals(myNXT.getText().toString())) {
            nxtDevice = bluetoothDevice;
            break;
         }
      }
 
      nxtBTsocket = bxtDevice.createRfcommSocketToServiceRecord(
                  UUID.fromString("00001101-0000-1000-8000-00805F9B34FB"));
      nxtBTsocket.connect();
      nxtDos = new DataOutputStream(nxtBTsocket.getOutputStream());
      nxtUno = new DataInputStream(nxtBTsocket.getInputStream());
   } catch (IOException e) {
      // Error en la creación de la conexión
   }
}
 
// Envío de datos por el canal de comunicación
public void sendNXTcommand(int command, int value){
    if (nxtDos == null) {
        return;
    }
    try {
        nxtDos.writeInt(command);
        nxtDos.writeInt(value);
        nxtDos.flush();
    } catch (IOException ioe) {
        // Error en la escritura
    }
}
 // Cerrar el canal de comunicación
private void destroyNXTConnection(){
    try {
        if (nxtBTsocket != null) {
            // send one close message
            sendNXTcommand(DISCONNECT,0);
            nxtBTsocket.close();
            nxtBTsocket = null;
        }
        nxtDos = null;           
    } catch (IOException e) {
        // Error cerrando la conexión
    }
}

Para el envío de las órdenes al robot se pueden incluir botones en la interfaz de nuestra aplicación Android y ligarlos con las acciones contenidas en el protocolo de comunicación.

No obstante, estas acciones también pueden ligarse con el movimiento del dispositivo Android mediante la utilización del sensor de aceleración. ¡Esto será el material de otro post en breve! De momento, os dejamos un vídeo del resultado final.
